\begin{abstract}
本文面向 NEURONCLAW 七日 Hackathon A 赛道 Paper-to-ARM 项目，在完全保留原有 ARM 九模块、命令行入口、Web 工作台、成功/失败双案例、基础工具与安全边界的前提下，提出并实现一组增量插件式增强模块。新增内容覆盖 LangGraph 状态机、PDF 原生解析、量化评测、安全审计、多文献融合、真实 PubMed 检索包装、依赖锁定、统一构建脚本与答辩材料生成。优化目标不是替换原系统，而是在稳定基线上闭环补齐六项工程短板。
\end{abstract}

\section{项目概述与原有能力}
原项目已经具备 A 赛道核心闭环：输入脑科学/ECS 论文，输出 metadata、claims、evidence、protocol、runbook、eval\_plan、provenance、limitations、artifacts 九模块 ARM 资产；支持 \texttt{main.py run/demo}、FastAPI Web 上传工作台、JSON/YAML 导出、trace 回放、成功案例 \texttt{brain\_ECS\_review.txt} 与残缺输入失败阻断案例。原有 \texttt{literature\_extract} 与 \texttt{reference\_validator} 保持 quote-only claim、DOI/PMID 风险标记、ECS 标签和医学科研边界。

\section{原系统现存6项短板逐条分析}
\subsection*{短板1：Agent编排轻量化，无LangGraph状态机、断点、人工审批、批量容错}
增量新增独立子包 arm\_agent/langgraph\_orchestrator，实现：1. 基于LangGraph重构可切换并行工作流（原有线性SDK流程保留作为默认分支，用户可选启用LangGraph增强版）；2. 全局Checkpoint持久化，论文批量解析中断后支持断点续跑；3. 人工复核暂停节点：当检测到结论冲突、引用缺失、高风险医学推断时自动暂停，输出人工审批表单，支持确认/驳回/补充输入后恢复流程；4. 批量任务隔离容错：单篇论文解析失败仅隔离输出独立失败报告，不阻塞整批任务，支持单文件重试机制；5. 状态全留存：每一步工具调用、提取结果、校验状态存入会话持久存储。

\subsection*{短板2：仅支持TXT，缺失PDF原生解析、图表/表格/OCR提取}
增量新增独立工具包 tools/pdf\_parser\_tools，新增可插拔插件：1. 基于pdfplumber+PyPDF2实现PDF全文、图注、表格原生解析，支持批量PDF输入；2. 简易OCR封装处理扫描版论文图表，提取图表文本内容绑定evidence溯源；3. 完善原有caption-only插件框架，实现真实图表证据绑定到对应claim；4. 输入层兼容PDF/TXT双格式，原有TXT解析逻辑完全不动，PDF作为新增输入分支。

\subsection*{短板3：评测体系只有定性单元测试，缺少ARM量化评测、幻觉检测、引用内容校验}
增量新增独立模块 arm\_agent/evaluation\_engine：1. 标准化专家标注数据集Schema，支持导入人工标注ARM标准样本；2. 量化评测指标自动计算：结论提取精确率、召回率、F1分数；3. 幻觉检测模块：逐句比对claim与原文文本，标记无原文支撑的模型生成内容，输出幻觉统计报表；4. 增强reference\_validator工具：新增原文内容匹配校验，不仅检查DOI/PMID存在性，还校验引用描述与论文原文一致性，拦截虚假引用；5. 新增评测专用pytest测试用例，不覆盖原有16个基础用例。

\subsection*{短板4：安全溯源体系薄弱，缺少完整审计、Prompt注入防护、精细化限流}
增量新增独立模块 arm\_agent/security\_audit：1. 全链路完整审计Trace日志：记录完整输入输出快照、模型版本、token消耗、API耗时、每一条claim完整生成链路，支持导出审计报告；2. 输入防护网关：文件格式校验、文件大小限制、Prompt Injection检测拦截逻辑；3. 精细化并发资源限流模块：区分单文件/批量文件API请求配额，动态控流避免DeepSeek接口报错；4. 新增安全校验前置中间件，串联在原有流程最前端，不改动原有业务代码。

\subsection*{短板5：工程细节粗糙，依赖锁、细分异常、前端交互、统一构建脚本}
增量优化，不改动原有业务逻辑：1. 生成 requirements-lock.txt 锁定全部依赖版本，解决openai-agents与cozepy版本冲突；2. 全局统一异常捕获分层：区分文件损坏、空文本、API超时、图表缺失、引用无效等细分异常，每种异常输出标准化失败ARM报告；3. 迭代Web前端页面 web/静态资源：新增冲突证据高亮、原文证据片段跳转、幻觉标记可视化交互面板；4. 新增统一跨平台构建脚本 build.sh / build.bat，统一环境初始化、测试、打包导出流程，清理冗余无用TeX/临时日志文件，保留必要文档TeX源码。

\subsection*{短板6：仅单篇独立ARM，无多文献融合、冲突对齐、外部真实文献检索}
增量新增独立工具包 tools/multi\_literature\_fusion：1. 多论文ARM合并模块：多篇同主题论文批量导入后，自动对齐术语、去重、标记矛盾结论，增加可信度加权字段存入ARM provenance；2. 重构literature\_search工具：从本地fixture模拟改为对接PubMed公开检索接口，实现真实外部文献检索，自动补充佐证证据绑定对应claim；3. 新增跨论文冲突检测子工具，冲突信息写入ARM limitations与审计日志。

\section{量化评测、安全审计与多文献融合}
量化评测通过专家标注 claim 集计算 precision、recall 与 F1；幻觉检测要求 claim 原文逐句出现在 source text 中，否则标记 review\_required。安全审计记录输入输出快照、模型版本、token、API 耗时与 claim lineage。多文献融合模块对 claims 按 raw text 去重，再调用冲突检测与可信度评分，将结果写入 provenance 与 limitations patch。

\section{局限性与未来规划}
新增 OCR 当前为显式 stub，避免夸大扫描版图像理解；PubMed 检索包装依赖网络，离线环境需回退模拟检索；LangGraph 增强版保留 fallback 状态机，后续可接入正式 graph 节点、人工审批 Web 表单和 C 赛道知识图谱导出。

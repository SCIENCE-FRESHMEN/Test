\begin{frame}{项目背景与ARM核心价值}
  \begin{itemize}
    \item 行业痛点：当前许多科研Agent仅完成浅层文本摘要，分散结论、图表、实验链路不存在结构化绑定，无法供下游AI自动复用复现，难以达到工业级Agent落地标准。
    \item 全量实现方案：本项目完整落地ARM九大结构化字段：metadata、claims、evidence、protocol、runbook、eval\_plan、provenance、limitations、artifacts，全字段无缺失，确保科研资产的结构化与完整性。
    \item 核心功能全覆盖：原生支持至多5篇脑科学论文批量输入、每条结论强制绑定原文溯源链路、内置Dry-run自动化试运行、标准化ARM资产导出与完整Trace回放，全面满足科研场景的严苛需求。
  \end{itemize}
  \begin{block}{核心竞争力}
    彻底脱离普通大模型摘要Demo的能力下限，产出可校验、可回放、可批量导出、原生无缝兼容知识图谱的标准化科研资产；整体分层架构、插件化设计、自动化测试体系完全对标开源仓库工程规范，具备极高的开源分发潜力，完美契合可复用、可复现的核心标准。
  \end{block}
\end{frame}

\begin{frame}{原有系统六大短板梳理}
  \begin{enumerate}
    \item Agent调度能力单薄：未引入LangGraph状态机，缺失断点续跑、人工审批、批量隔离容错机制，多论文长流程任务稳定性不足，难以满足工业级开源项目标配的调度能力。
    \item 文档解析能力存在局限：仅兼容纯文本TXT输入，缺失PDF原生解析、图表/表格/OCR证据绑定模块，无法适配真实科研论文的主流格式，落地场景受限。
    \item 输出质量管控仅依靠定性约束：缺少ARM提取量化评测、幻觉自动检测、引用原文一致性自动化校验模块，无法量化输出可信度，缺乏成熟开源工程必备的自动化质量检测组件。
    \item 安全溯源体系轻量化：缺少全链路审计日志、Prompt注入防护、精细化API限流管控模块，无法达到完整安全审计交付规范，难以满足医学科研数据的安全约束要求。
    \item 工程落地细节待优化：无固定依赖锁定机制、异常捕获粒度粗放、前端交互功能单薄、缺少一键构建脚本，代码复现门槛较高，需进一步向开源仓库标准化交付规范靠拢。
    \item 仅支持单篇论文独立生成ARM资产，不存在多文献融合、矛盾结论自动对齐、外部PubMed真实检索联动能力，无法支撑多源证据交叉验证等高阶科研场景。
  \end{enumerate}
\end{frame}
